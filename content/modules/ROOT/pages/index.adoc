Welcome to the *AgentOps in Production* workshop!

Multi-agent AI systems fail in ways that monolithic applications never do.
A single slow agent can poison the entire pipeline.
Opaque token costs can blow through your budget before anyone notices.
Silent retrieval failures return empty results without raising an exception.
Standard monitoring catches none of it.

This workshop gives you the observability discipline to find those failures before your users do.

== What you will learn

In this workshop, you will:

* Identify the components of a multi-agent AI architecture and explain why distributed observability matters for production reliability
* Navigate Grafana dashboards to analyze AI-specific metrics -- token usage, inference latency, and tool call success rates
* Write LogQL queries in LokiStack to filter and correlate logs across a running multi-agent application
* Instrument a LangGraph application with MLflow autolog for zero-code-change distributed tracing
* Analyze trace timelines to identify latency bottlenecks and silent tool failures in multi-agent workflows
* Run prompt evaluations, compare versions, and detect regressions before they reach production
* Package evaluations into a schedulable AI Pipeline on OpenShift AI

== Who this is for

This workshop is designed for *platform engineers, AI platform administrators, and MLOps leaders* who are responsible for keeping AI workloads reliable in production.

*Experience level*: Intermediate -- you should be comfortable with the OpenShift console and Python at a reading level.
No prior MLflow, LangGraph, or Grafana experience is required.

== Prerequisites

Before starting, you should have:

* Basic OpenShift or Kubernetes familiarity -- you can navigate to a namespace and view pod logs
* Python at a reading level -- you can understand imports, function calls, and dictionaries
* General awareness of LLMs -- you know what a prompt is, what tokens are, what inference means

== Workshop environment

Everything is already deployed and running when you arrive.
You will not install or configure anything -- your job is to observe and analyze a live system.

Your environment includes:

* *AI Ops Assistant* -- a four-agent LangGraph application handling live traffic
* *Grafana* -- pre-built AI observability dashboard connected to Prometheus and Loki
* *LokiStack* -- collecting logs from all workloads in the `{project_namespace}` namespace
* *MLflow* -- distributed tracing and Prompt Registry, tracking URI pre-configured
